# Шаг 0003: Разбор ошибки `prisma init`

## Команды терминала

```bash
npx prisma init --bd --output ../src/generated/prisma
```

```bash
# корректный вариант для текущего проекта
npx prisma init --db --output ./src/generated/prisma
```

## Что произошло

- Prisma CLI не выполнил инициализацию и показал help-экран.
- Причина: неверный флаг `--bd` (в CLI существует `--db`).

## Важная поправка по пути

- Введенный путь `../src/generated/prisma` указывает на папку вне репозитория (`C:/work/src/...`).
- Для проекта `C:/work/TS_PostgreSQL` нужен путь внутри репозитория:
  - `./src/generated/prisma`

## Плюсы шага

- Получен реальный кейс чтения CLI-ошибок: help-вывод сразу показывает корректные флаги.
- Зафиксировано правило: при ошибке параметров сначала сверяемся с `Usage` и `Options`.

## Минусы и риски

- Если запустить с `../`, артефакты генерации уйдут в соседнюю директорию и сломают структуру проекта.
- Ошибка одного символа во флаге блокирует весь шаг и может сбить темп обучения.

## Что делать дальше

- Запустить корректную команду:

```bash
npx prisma init --db --output ./src/generated/prisma
```

- После успешного запуска зафиксировать новый коммит и отдельно разобрать:
  - созданные файлы,
  - содержимое `schema.prisma`,
  - изменения в `.env`.

## Комментарий к объяснению из видео

В видео такие ошибки обычно не показывают, но на практике это нормальная часть работы.
Главное не угадывать, а читать help-вывод инструмента и сразу корректировать флаг/путь.
