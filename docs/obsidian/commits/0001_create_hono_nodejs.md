# Шаг 0001: Создание каркаса Hono-приложения

## Что сделано

- Выполнен `bun create hono@latest`.
- Выбран template: `nodejs`.
- Подтверждена работа в непустой директории (`TS_PostgreSQL`).
- Установлены зависимости через `bun`.
- Создан и зафиксирован коммит `create hono` (`cd1ba30`).

## Какие файлы добавились

- `.gitignore`
- `README.md`
- `package.json`
- `bun.lock`
- `tsconfig.json`
- `src/index.ts`
- файлы документации в `docs/obsidian/...`

## Что это значит технически

- Проект переведен из "пустого репозитория" в минимально запускаемое backend-приложение.
- Сервер уже слушает порт `3000` и отдает ответ `Hello Hono!` по `/`.
- TypeScript-сборка настроена (`tsc` -> `dist/`), запуск в dev-режиме через `tsx watch`.
- Используется ESM (`"type": "module"`) и режим `module: "NodeNext"`.

## Плюсы решения

- Очень быстрый старт: рабочий сервер получается за 1 команду.
- Совместимость с TypeScript и современным ESM уже настроена шаблоном.
- Низкий порог входа: можно сразу переходить к Prisma и PostgreSQL, не тратя время на ручной bootstrap.
- Наличие lock-файла (`bun.lock`) фиксирует версии зависимостей и улучшает воспроизводимость.

## Минусы и риски

- В `README.md` команды показаны через `npm`, хотя проект создан под `bun`; это может путать.
- В `.gitignore` есть исключения для `.vscode/launch.json` и snippets: есть риск случайно закоммитить локальные IDE-настройки.
- Версии шаблона и зависимостей "плавающие" (`^`), при повторе через время результат может немного отличаться.
- Шаблон пока не содержит проверок качества (lint/test/format), значит регрессии легче пропустить.

## Что улучшить на ближайшем шаге

- Привести `README.md` к `bun`-командам для единообразия.
- Добавить базовые скрипты качества (`lint`, `test`), если это не конфликтует с темпом видео.
- Зафиксировать этап Prisma отдельным коммитом: установка CLI, init schema, env с `DATABASE_URL`.

## Комментарий к объяснению из видео

В видео этот этап обычно показывают как "просто генерацию шаблона", но здесь важный практический момент:
ошибка `Failed to fetch ... api.github.com` была связана с включенным VPN.
То есть проблема была не в Hono/Prisma, а в сетевом маршруте к GitHub API.
Это типичный реальный кейс, который редко проговаривают в обучающих роликах.
